## 📋 项目概述

**草莓管家** 是一款专为草莓种植者打造的 AI 智能问答助手，提供专业的草莓种植知识、病虫害防治、栽培技术等专业咨询。

### 项目定位
- **目标用户**：草莓种植者、农业技术人员、农业爱好者
- **核心价值**：结合大语言模型 + 专业知识库，提供准确、实用的草莓种植指导
- **竞争优势**：针对性强、操作简单、多平台覆盖、智能交互

### 核心功能
- 🍓 **专业问答**：基于草莓种植知识的 AI 对话
- 📚 **知识库增强**：GraphRAG 检索增强，提升回答准确性
- 🎤 **多模态输入**：文字 + 语音（待接入）两种交互方式
- 🔊 **语音播报**：AI 回答语音朗读，方便田间操作
- 📱 **多端适配**：支持 H5、App、小程序等多个平台
- ⚙️ **个性化配置**：字体大小、语音开关等自定义设置

└──────┬────────────────┬─────────────────┘
       │                │

┌─────────────┐  ┌─────────────────────┐

### 技术栈选择

| 组件 | 技术选择 | 理由说明 |
|------|----------|----------|
| **前端框架** | UniApp + Vue 3 + Vite | 一套代码多端发布，覆盖移动端全场景 |
| **后端服务** | Node.js + Express | JavaScript 全栈，开发效率高 |
| **数据库** | MySQL | 关系型数据，支持复杂查询 |
| **AI 模型** | DeepSeek API | 高性价比，支持流式响应 |
| **知识库** | GraphRAG HTTP API | 检索增强，提升专业领域准确性 |
| **语音播报** | 系统 TTS | 原生实现，兼容性好 |
| **语音识别** | 讯飞 ASR（规划） | 中文识别准确率高 |
| **状态管理** | Pinia | Vue 3 官方推荐状态管理方案 |

### 数据流设计

```
用户提问 → 知识库检索 → 构建系统提示词 →
DeepSeek 流式对话 → 实时渲染 → 存储记录 →
语音播报（可选）
```

## 📡 API 接口设计

### 接口规范
- **Base URL**: `http://localhost:3000/api`（开发环境）
- **认证方式**: Bearer Token (JWT)
- **响应格式**: 统一 JSON 响应

### 接口列表

#### 1. 认证模块 (`/api/auth`)
- `POST /auth/register` - 用户注册
- `POST /auth/login` - 用户登录
- `POST /auth/refresh` - Token 刷新（规划）

#### 2. 聊天模块 (`/api/chat`)
- `POST /chat/stream` - 流式对话（SSE）
- `GET /chat/history` - 获取对话历史（规划）

#### 3. 知识库模块 (`/api/knowledge`)
- `POST /knowledge/query` - 知识库检索
- `GET /knowledge/status` - 知识库状态检查

#### 4. 历史记录模块 (`/api/history`)
- `GET /history` - 获取对话历史列表
- `DELETE /history` - 清空对话历史
- `DELETE /history/:id` - 删除单条记录

### 接口示例

**流式对话请求**
```json
POST /api/chat/stream
Authorization: Bearer eyJhbGci...
Content-Type: application/json

{
  "messages": [
    {"role": "user", "content": "草莓白粉病怎么防治？"},
    {"role": "assistant", "content": "草莓白粉病...（历史回答）"}
  ]
}
```

**流式响应（SSE）**
```
data: {"delta": "草", "index": 0}
data: {"delta": "莓", "index": 1}
data: {"delta": "白", "index": 2}
...
data: [DONE]
```

## 🚀 部署指南

### 1. 服务器要求
- **操作系统**: Ubuntu 22.04 LTS 或 CentOS 7+
- **配置**: 2核 4G 内存，50GB 硬盘
- **网络**: 公网 IP，80/443 端口开放

### 2. 后端部署
```bash
# 克隆代码
git clone <repository>
cd backend

# 安装依赖
npm install --production

# 配置环境变量
cp .env.example .env
vim .env  # 编辑生产环境配置

# 使用 PM2 进程守护
npm install -g pm2
pm2 start src/app.js --name strawberry-backend
pm2 save
pm2 startup
```

### 3. Nginx 配置
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;

        # SSE 流式响应必须配置
        proxy_buffering off;
        proxy_read_timeout 300s;
    }

    # 前端静态文件（H5版本）
    location / {
        root /var/www/strawberry-frontend;
        try_files $uri $uri/ /index.html;
    }
}
```

### 4. 前端打包
```
HBuilderX → 发行 → 选择目标平台
```

**各平台打包注意事项**：
- **App**: 配置应用签名，申请相应权限
- **小程序**: 配置 appid，上传微信开发者工具
- **H5**: 修改 `utils/request.js` 中的 BASE_URL

### 5. SSL 证书（生产环境必须）
- 使用 Let's Encrypt 免费证书
- 配置 HTTPS 重定向
- 确保 API 请求走 HTTPS

## 🔐 安全策略

### 1. 认证安全
- **JWT 令牌**: 有效期 7 天，使用强密钥
- **密码存储**: bcrypt 哈希 + 盐值
- **登录限制**: 失败次数限制（规划）

### 2. API 安全
- **CORS 配置**: 仅允许信任域名
- **请求限流**: 防止恶意请求
- **输入验证**: 所有参数严格验证

### 3. 数据安全
- **数据库连接**: 使用连接池，防止连接泄露
- **敏感数据**: API Key 等敏感信息不落库
- **日志记录**: 不记录敏感信息

### 4. 前端安全
- **本地存储**: 不存储敏感信息
- **代码混淆**: 生产环境代码混淆
- **依赖检查**: 定期更新依赖，修复安全漏洞

## 📞 支持与维护

### 1. 常见问题

**Q: H5 环境跨域问题**
A: 开发时配置代理，生产环境配置正确的 CORS 规则。

**Q: 语音播报不工作**
A:
1. 检查浏览器/设备是否支持 Web Speech API
2. 检查音量设置
3. App 环境检查权限配置

**Q: API Key 泄露风险**
A: 生产环境务必使用后端中转方案。

### 2. 故障排查流程
```
1. 检查网络连接
2. 检查控制台错误日志
3. 验证 API Key 和配置
4. 测试基础功能（如知识库连接）
5. 检查存储空间和权限
```

### 3. 性能监控（规划）
- **前端性能**: Lighthouse 评分，页面加载时间
- **后端性能**: API 响应时间，错误率
- **用户行为**: 功能使用频率，用户留存
